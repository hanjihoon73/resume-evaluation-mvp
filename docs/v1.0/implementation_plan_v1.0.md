# 이력서 평가 MVP 구현 계획서

본 문서는 `resume-evaluation-mvp-planning.md` 기획안의 내용을 100% 반영하여 작성된 구현 계획서입니다. 모든 수치, 기준, 화면 구성을 기획서와 동일하게 구현하는 것을 원칙으로 합니다.

## 1. 프로젝트 개요 및 환경 설정
- **기본 스택**: TypeScript, Next.js (App Router)
- **UI/UX**: Tailwind CSS, shadcn/ui, Lucide React icons
- **디자인 컨셉**: 트렌디한 다크 모드, 네온 스타일, 미니멀리즘, 글래스모피즘
- **메인 컬러**: 블루 계열 / **보조 컬러**: 오렌지, 레드 계열
- **폰트**: Pretendard (웹 및 PDF 공통)

## 2. 상세 평가 로직 (기획안 수치 100% 반영)

### 항목별 상세 평가 기준
1. **관련 전공 (10%)**: 최소 0점, 최대 100점
   - CS 관련 전공(+60), 학점 가산점(4.0+ 20, 3.5+ 10, 3.0+ 5), 석사(+10), 박사(+10), 비전공(최대 50)
2. **근속 기간 (20%)**: 최소 0점, 최대 100점
   - 4년 이상 필수(미달 시 1년당 -10), 최종 근속(2년 40, 3년 50, 4년 60), 평균 근속(2년 30, 3년 40), 이직 사유(+10)
3. **기술 스택 (40%)**: 최소 0점, 최대 100점
   - Backend(60): Java/Kotlin(15), Spring Boot(10), DB(10), JPA(5), Test(5)
   - Infra(20): AWS(20), 기타 Cloud(10)
   - Frontend(20): React, JS, TS, Next.js (각 5)
4. **AI 역량 (20%)**: 최소 0점, 최대 100점
   - 기본 사용(30), 개발 사례(40), Prompt Engineering(20), 마인드셋(10)
5. **컬처핏 (10%)**: 최소 0점, 최대 100점
   - 자율성(30), 열정(25), 효율성(25), 협업(20)

### 분석 엔진 구현 정책
- **모델**: Gemini 3 Flash Preview (`models/gemini-3-flash-preview`)
- **실행 방식**: 모든 요청은 **직렬(Serial)**로 처리 (n번 파일 완료 후 n+1번 분석)
- **반복 분석**: 파일 1개당 평가 항목별 총 3회 분석 후 평균 점수 산출 -> 배점 비중 적용

## 3. 화면별 구현 상세

### 홈 화면
- **파일 업로드**: 다중 파일 선택, 파일명 형식(`포지션_이름_지원채널.pdf`) 파싱
- **정렬 로직**: 지원채널 > 포지션 > 이름 순 오름차순
- **기능**: [분석 시작] 버튼 (업로드 후 활성화), 분석 진행 프로그래스바
- **결과 목록**: 연번, 이름, 포지션, 지원채널, 총점, 적합도(라벨링), [리포트 확인] 버튼

### 리포트 화면
- **버튼**: [PDF 다운로드] (리포트_포지션_이름_지원채널.pdf), [목록으로 가기] (결과 영역 자동 포커싱)
- **기본 정보**: 이름, 포지션, 지원채널, 최종학력(학사~박사), 학점, 나이(현재연도-생년), 경력기간
- **종합 평가**: 총점, 적합도(4단계 등급), 총평(200~500자), 장단점 구분
- **세부 평가**: 5개 항목 카드(점수, 별점 1~5개, 내용 200~500자, 인터뷰 체크사항)

## 4. 기술적 도전 과제 및 해결 방안
- **PDF 파싱**: `RegExp`를 활용하여 파일명에서 포지션/이름/지원채널 정확히 추출
- **PDF 생성**: `@react-pdf/renderer`에 Pretendard 폰트 임베딩 및 기획안 레이아웃 구현
- **상태 관리**: 분석 중인 파일의 진행률 및 결과를 관리하기 위한 전역/로컬 상태 설계
