# 리포트 UI 개편 및 전공 항목 추가 계획 (v1.5)

리포트 화면의 가독성을 높이고, 지원자의 전공 정보를 추가하여 평가의 깊이를 더합니다.

## 1. 데이터 구조 및 AI 엔진 수정
- **목표**: 지원자 전공(Major) 정보를 추출하고 시스템 전반에서 활용 가능하도록 업데이트합니다.
- **수정 파일**:
    - [MODIFY] `src/lib/analysis-engine.ts`: `AnalysisResult` 인터페이스의 `basicInfo`에 `major` 필드 추가
    - [MODIFY] `src/lib/system-prompt.ts`: AI에게 전공 정보를 추출하여 JSON으로 반환하도록 프롬프트 업데이트
- **검증**: 분석 결과 JSON에 전공 정보가 포함되는지 확인

## 2. 리포트 UI (`report/[id]/page.tsx`) 개편

### [STEP 1] 상단 제목 섹션 단순화
- 제목 하단의 포지션, 이름, 지원채널 정보를 제거합니다. (기본 정보 섹션으로 통합 예정)

### [STEP 2] 기본 정보 섹션 재구성
- **표시 순서**: 포지션 > 이름 > 지원채널 > 전공 > 최종학력 > 학점 > 나이 > 경력기간
- **레이아웃**: 기존의 세로형 카드에서 **가로 한 행(Horizontal Row)**으로 모든 정보를 나열하여 공간 효율성을 높입니다.
- 전공 정보가 없을 경우 "정보 없음"으로 표시합니다.

### [STEP 3] 종합 점수 및 총평 섹션 분리
- **레이아웃**: 종합 점수(적합도 포함)와 총평을 각각 별개의 카드로 분리하고, 한 행에 나란히 배치합니다.

### [STEP 4] 상세 항목 분석 레이아웃 변경
- **레이아웃**: 현재 그리드 형태(2~3열)인 상세 항목(전공, 경력 등) 분석 카드를 **한 행에 하나씩(Full Width)** 배치하여 내용을 더 자세히 읽을 수 있도록 합니다.

## 3. 검증 계획

### 수동 확인 사항
- 새로운 기본 정보 순서와 레이아웃이 디자인 시스템과 조화를 이루는지 확인
- 상세 분석 항목이 한 행씩 표시될 때 텍스트가 잘리지 않는지 확인
- 전공 정보가 실제 이력서 내용에 맞게 추출되는지 테스트 (Gemini/Claude 모두 확인)

---
> [!IMPORTANT]
> 기본 정보가 한 행으로 표시될 때 항목이 많으므로(8개), 텍스트 크기 및 간격을 조절하여 가독성을 유지하는 것이 핵심입니다.
